【OptiCore 新增 UI 表单输入功能】

当前程序仅支持通过上传 JSON 文件来输入镜头参数。现需在 Streamlit 界面增加第二种输入方式：**用户直接在页面上填写表单**，点击按钮后生成计算结果。

请基于现有代码架构进行增量修改，不要重写整个项目。

---

## 一、UI 布局要求

在 Streamlit 界面中，按以下方式组织：

### 1. 输入方式切换
在页面顶部添加一个单选按钮或选项卡，让用户选择输入方式：
- “📂 上传 JSON 文件”（保留现有功能）
- “✏️ 手动填写参数”（新增功能）

### 2. 手动填写表单（当用户选择“手动填写”时显示）

表单按以下分组展示，每个字段都要有清晰的标签和默认值：

**系统参数组：**
- 物距 (mm)：数字输入框，默认 -500（输入 0 或 inf 表示无限远）
- 入瞳半径 (mm)：数字输入框，默认 10
- 光阑所在面索引：数字输入框，默认 0
- 最大视场像高 (mm)：数字输入框，默认 12

**计算设置组：**
- 波长选择：多选框，选项为 d、F、C，默认全选
- 视场模式：下拉选择，选项为 angle / object_height / image_height，默认 image_height
- 视场数值：数字输入框，默认 12
- 孔径角 (度)：数字输入框，默认 5
- 孔径分数：多选或输入框（如 0.0, 0.7, 1.0），默认这三个值

**光学面列表组：**
- 提供一个“添加面”按钮，点击后在下方新增一行输入框
- 每一行包含：曲率半径 (mm)、厚度 (mm)、玻璃名称、通光半口径 (mm)
- 默认预置两个面（单透镜示例），用户可修改或增减
- 提供“删除此面”按钮，方便移除不需要的面

**玻璃库设置：**
- 玻璃库文件名：文本输入框，默认 "CDGM-ZEMAX202111.AGF"
- 或提供一个下拉菜单，内置常用玻璃库名称

**操作按钮：**
- “开始计算”按钮

## 二、数据流逻辑

当用户点击“开始计算”时：
1. 从表单中收集所有参数，**组装成一个与 JSON 文件结构完全相同的 Python 字典**（即 system_parameters、calculation_settings、surfaces、glass_library）。
2. 将该字典传入现有的计算引擎（与 JSON 上传后调用的计算函数是同一个）。
3. 计算结果按现有方式展示（保持与 JSON 输入一致）。

**核心要求**：两种输入方式（JSON 上传和表单填写）最终必须调用**同一个核心计算函数**，避免逻辑分支导致结果不一致。

---

## 三、代码实现要求

### 1. 复用现有代码
- 不要重写 LensLoader 或核心计算引擎。
- 在 app.py 中新增一个函数 `build_lens_from_form(form_data)`，将表单数据转换为 Lens 对象或字典。
- 现有 JSON 上传逻辑保持不变。

### 2. 动态表单
- 光学面列表使用 `st.session_state` 管理，确保添加/删除面时页面刷新后数据不丢失。
- 每个面至少包含：radius、thickness、glass、diameter 四个字段。
- 面的数量可以动态增减（至少保留 1 个面）。

### 3. 表单验证
- 数字字段做基本类型校验（如半径不能为空，且必须是数字）。
- 如果必填字段为空，点击“开始计算”时给出友好的提示。

### 4. UI 风格
- 与现有 Streamlit 界面风格保持一致。
- 使用 `st.expander` 对参数分组（可选）。
- 输入框使用 `st.number_input` 或 `st.text_input`，根据字段类型选择。

---

## 四、验收标准

1. 用户选择“手动填写参数”后，能看到完整的表单。
2. 填写完成后点击“开始计算”，能正常输出结果（与 JSON 上传的结果一致）。
3. 切换回“上传 JSON 文件”后，原有功能完全不受影响。
4. 添加/删除光学面时，页面能正确更新，不会丢失其他已填数据。
